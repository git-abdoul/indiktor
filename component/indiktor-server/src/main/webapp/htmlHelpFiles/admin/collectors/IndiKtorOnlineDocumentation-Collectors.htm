COLLECTORS - Introduction

What is a Collector?

A Collector is a dedicated IndiKtor configuration specifying the customizing the metrics you wish to monitor.

A Collector is a logical organization of metrics, relying upon a physical connection (Connector) previously defined.

What is a collector used for?

Collectors are set up by users with Admin rights to provide a logical organization of metrics that will then be used for alerting purposes. A connector provides a physical connection to the underlying information system: database, server, network, or Software. A collector provides the link between the data collected and the physical connection to that data type.

Who uses Collectors?

In essence, any IndiKtor user will interact with collectors (with or without knowing it) since any metric managed in IndiKtor comes though a collector. In terms of management, only users with Admin rights are entitled to create, modify or delete collectors.

COLLECTORS -Filter Collector

Collectors can be organized by Logical Environment. The filtering options enables users to view collector data limited to one specific Logical Environment. To filter collectors:

  1. Click on the filter icon
  2. Select from the drop-down list, the logical environment you wish to use

  1. By default, the environment selection is set to ALL.
  2. Once you have selected a logical environment, click   to close and go back to the selected list of collectors

 

COLLECTORS -Create Collector

To create a new collector, you will need to define its Properties and Scheduling

Collector Properties are designed to provide all required details on how the data needs to be fetched and organized for this specific collector.

Collector Scheduling is designed to define rate and time at which the metric details need to be fetched for a given collector.
